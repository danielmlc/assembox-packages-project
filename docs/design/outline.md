## 一、项目概述

### 1.1 平台定位

企业级 SaaS 低代码开发平台，以模型为核心，通过可视化配置和元数据驱动，自动生成代码并发布部署。

**面向用户**：
- 业务人员：通过可视化设计器快速搭建应用
- 开发人员：通过代码生成提高开发效率

### 1.2 核心理念

- **元数据驱动**：配置即代码，通过元数据配置生成可运行的应用
- **组件化管理**：以组件为最小粒度进行配置和构建
- **多租户支持**：支持系统、全局、租户三层配置继承
- **自动化发布**：构建生成完整项目代码，走 CI/CD 自动部署

### 1.3 业务组织结构

```
业务块（微服务）
  └── 模块（逻辑分组）
       └── 组件（最小配置单元）
            ├── 服务端：Controller, Service, Entity, Repository
            └── 前端：Form, Table, Filter, Export, Print
```

## 二、总体架构

### 2.1 架构分层

```
┌─────────────────────────────────────────┐
│   低代码设计器层                         │  ← 可视化配置界面
│   - 元数据建模                           │
│   - 组件配置                             │
│   - 版本管理                             │
├─────────────────────────────────────────┤
│   前端层                                 │  ← 生成的前端应用
│   - 页面路由                             │
│   - 业务组件                             │
│   - 数据交互                             │
├─────────────────────────────────────────┤
│   服务层                                 │  ← 运行时服务
│   - 运行时配置                           │
│   - 配置继承                             │
│   - 业务逻辑                             │
├─────────────────────────────────────────┤
│   构建发布层                             │  ← 代码生成与部署
│   - 快照管理                             │
│   - 代码生成                             │
│   - CI/CD 流水线                         │
├─────────────────────────────────────────┤
│   存储层                                 │  ← 持久化存储
│   - 数据库（元数据）                     │
│   - OSS（配置内容）                      │
│   - Git（配置版本）                      │
│   - Gitea（代码版本）                    │
└─────────────────────────────────────────┘
```

### 2.2 核心流程

**配置流程**：
```
设计器配置 → 保存元数据（DB + OSS） → 版本管理（Git） → 构建触发
```

**构建流程**：
```
触发构建 → 创建快照 → 获取配置 → 生成代码 → 推送 Gitea → CI/CD → 部署
```

**运行流程**：
```
运行时请求 → 配置继承（租户 > 全局 > 系统） → 业务逻辑 → 返回响应
```

## 四、核心设计决策

### 4.1 配置管理

| 配置类型 | 层级 | 用途 | 生成代码 |
|---------|------|------|---------|
| 运行时配置 | 系统 → 全局 → 租户 | 动态配置 | 否 |
| 构建配置 | 系统 → 租户 | 静态代码 | 是 |

### 4.2 版本管理

| 版本类型 | 粒度 | 管理方式 |
|---------|------|---------|
| 产品版本 | 模块独立版本 | 保留历史，支持回滚 |
| 租户版本 | 基于产品版本分支 | 独立迭代，不自动同步 |

### 4.3 存储分工

| 存储 | 内容 | 用途 |
|-----|------|------|
| 数据库 | 配置元信息 | 结构、关系、状态 |
| OSS | 配置内容 | JSON/YAML 文件 |
| Git | 原始配置 | 配置版本追溯 |
| Gitea | 生成代码 | 代码备份管理 |

## 六、设计原则

1. **简单清晰**：避免过度设计，代码只体现关键逻辑
2. **组件化**：以组件为最小粒度进行管理
3. **配置驱动**：通过配置生成代码，而非手写代码
4. **多租户**：支持租户级别的定制和独立迭代
5. **自动化**：构建、测试、部署全流程自动化
